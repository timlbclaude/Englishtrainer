☁️ Sync einrichten

Handy und Laptop sollen denselben Lernfortschritt zeigen. Einmal einrichten, dann läuft es automatisch.

Wie der Sync funktioniert

Der Trainer speichert pro Wort die Stufe (0–7), das nächste Fälligkeitsdatum und die Trefferquote. Diese Daten werden als einzelne JSON-Datei in einem privaten GitHub-Gist abgelegt. Bei jeder Bewertung pusht die App den neuen Stand in den Gist; beim Öffnen oder Zurückkehren auf einen Tab zieht sie den neuesten Stand. Beide Geräte nutzen denselben Gist – damit sind sie immer synchron.

1Token bei GitHub erstellen

Der Token ist ein Passwort, mit dem die App auf deinen Gist zugreifen darf. Er braucht nur Gist-Zugriff – kein Zugriff auf deine Repos.

  1. Öffne github.com/settings/personal-access-tokens/new (Fine-grained Token).
  2. Token name: English-Trainer-Sync
  3. Expiration: wähle Custom → ein Datum in 1 Jahr. (Erinnerung: rechtzeitig vor Ablauf neuen Token erstellen.)
  4. Resource owner: dein eigener Account (timlbclaude).
  5. Repository access: Public Repositories (read-only) — Gists werden separat behandelt, die App braucht keine Repo-Rechte.
  6. Account permissions → Gists → Access: Read and write. Das ist die einzige Berechtigung, die du setzen musst.
  7. Klicke Generate token. Der Token wird einmalig angezeigt — sofort kopieren. Er beginnt mit github_pat_.
Wichtig: Der Token ist ein Passwort. Nicht in Chats posten, nicht in Screenshots zeigen. Der Token wird nur in localStorage deines Browsers gespeichert und bei jedem Sync direkt an GitHub geschickt — er landet nirgendwo sonst.
Alternative – Classic Token: Wenn der Fine-grained-Flow zickt, funktioniert auch ein klassischer Token unter github.com/settings/tokens/new mit einzigem Scope gist.

2Auf dem ersten Gerät einrichten (z.B. Laptop)

  1. Öffne den English Trainer.
  2. Gehe zum Tab 🧠 Training.
  3. Oben rechts im Dashboard siehst du den Button ☁️ Sync einrichten. Klick drauf.
  4. Füge den gerade kopierten Token ins Feld GitHub Personal Access Token ein.
  5. Das Feld Gist-ID lässt du leer. Klick Speichern & Verbinden.
  6. Die App legt automatisch einen privaten Gist an und zeigt dir die neu erzeugte Gist-ID (sieht aus wie a1b2c3d4e5f6...).
  7. Mit dem Button 📋 ID kopieren kommt die ID in die Zwischenablage.
✓ Auf dem Badge oben rechts sollte jetzt ☁️ Synchronisiert · HH:MM stehen.

3Zweites Gerät koppeln (z.B. Handy)

  1. Öffne den English Trainer auf dem zweiten Gerät.
  2. Tab 🧠 Training → ☁️ Sync einrichten.
  3. Trag denselben Token ein (am einfachsten: Token vorher kurz in einer Notiz an sich selbst verschicken, dort auf dem Handy öffnen, kopieren, in App einfügen, Notiz löschen).
  4. Trag die Gist-ID vom ersten Gerät ein.
  5. Klick Speichern & Verbinden.
  6. Das Handy zieht jetzt den aktuellen Stand und übernimmt ihn.
Tipp für den Token-Transfer: E-Mail an dich selbst mit Betreff "ETA-Sync Token". Schreib den Token ins Mail-Body, nicht in den Betreff. Nach erfolgreichem Setup auf dem Handy: Mail löschen (auch aus dem Papierkorb). Alternativ: Token nochmal neu erstellen für das zweite Gerät – beide funktionieren parallel.

Wann wird synchronisiert?

  • Automatisch nach jeder Bewertung (✓ Gewusst / ✗ Nicht gewusst) – nach 1,5 s debounced, damit schnelle Klick-Serien zu einem einzigen Upload zusammengefasst werden.
  • Beim Öffnen der Seite – holt den aktuellsten Stand vom anderen Gerät.
  • Wenn der Tab wieder in den Vordergrund kommt – zum Beispiel nach dem Umschalten vom Handy zurück zum Trainer.
  • Wenn das Internet zurückkehrt nach kurzer Offline-Phase.
  • Manuell über den ⟳ Sync-Button neben dem Badge.

Konflikt-Auflösung

Falls doch mal beide Geräte gleichzeitig dasselbe Wort bewerten: Pro Wort-ID gewinnt der Eintrag mit dem jüngeren Zeitstempel (updatedAt, Millisekunden-Präzision). Das wird in der Praxis nicht auftreten, weil du alleine trainierst.

Sicherheit & Privatsphäre

  • Der Gist ist privat — nur mit deinem GitHub-Account sichtbar.
  • Der Token hat nur Gist-Scope. Selbst bei einem Leak kann damit niemand in deine Repos schreiben.
  • Token und Gist-ID liegen nur in localStorage deines Browsers — nicht im Repo, nicht in GitHub Pages.
  • Widerruf: Jederzeit unter github.com/settings/tokens den Token sofort invalidieren.

Trennen oder Zurücksetzen

Klick auf das Sync-Badge → Modal öffnet sich → Token-Feld leeren → Speichern & Verbinden. Das trennt das Gerät vom Sync. Der Gist selbst bleibt erhalten.

Um den Gist ganz zu löschen: gist.github.com öffnen, den Gist mit dem Titel „English Trainer – SRS Sync" aufrufen, Delete. Dann auf beiden Geräten im Sync-Modal auch das Gist-ID-Feld leeren.
